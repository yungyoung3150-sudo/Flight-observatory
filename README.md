# 마카오 항공 요금 방어 모니터 (macau-fare-monitor)

예스트래블 마카오 공동구매 항공권의 **요금 방어(defense) 모니터링** 로직을, 운영 중이던
구글 스프레드시트(`마카오 항공 최저가 모니터링`)에서 추출해 **검토 가능한 코드**로 옮긴 것이다.

- 원본 시트: 마카오 항공 최저가 모니터링 (예스트래블) — owner `miles@yestravel.co.kr`
- 수집 기준일: 2026-06-19 / 모니터링 구간: **2026-06-20 ~ 2026-10-31 (134일)**
- 대상 상품: 에어마카오 2박3일·3박4일, 에어부산 2박3일·3박4일

## 1. 원본 스프레드시트 분석

시트는 7개 탭으로 구성된 **"수집 → 비교 → 방어 판정 → 경보"** 파이프라인의 결과물이다.

| 탭 | 역할 | 핵심 컬럼 |
| :-- | :-- | :-- |
| `항공_최저가` | 에어마카오 시장 최저가(1인) 원시 수집 | 2박3일·3박4일 최저(1인), 수집일 |
| `에어마카오_2박3일` | **방어 시트** | 우리요금(2인), 시장최저(2인), 방어, 비고 |
| `에어마카오_3박4일` | **방어 시트** | 〃 |
| `에어부산_2박3일` | 방어 시트(시장최저 수집 예정) | 우리요금(2인)만 일부 입력 |
| `에어부산_3박4일` | 방어 시트(시장최저 수집 예정) | 〃 |
| `출발일`(통합) | 정제된 시장 최저가(1인) ISO 날짜판 | 2박3일·3박4일 최저(1인), 비고 |
| 수집 로그 | 수집 신선도 경보 | 수집시각, 데이터 신선도, `⚠ 데이터 미갱신` |

### 핵심 비즈니스 로직 — '방어'

각 출발일에 대해, 우리 공동구매가가 시장 최저가보다 **싸거나 같으면 방어 성공**, 비싸면 실패.
모든 비교는 **2인 합산가(KRW)** 기준이며, `시장최저(2인) = 시장 최저가(1인) × 2` 로 산출돼 있다.

```
gap = 시장최저(2인) - 우리요금(2인)
  gap >= 0  →  ✅ OK            (방어 성공, gap = 여유분)
  gap <  0  →  🔴 방어실패 {gap}  (예: "🔴 방어실패 -29,332", 부족분 = -gap)
  우리요금 또는 시장최저가 없으면 → 빈 칸(판정 불가)
```

### 발견한 데이터 품질 이슈 (CPO 검토 참고)

1. **`항공_최저가` 탭의 3박4일 열 포맷 깨짐** — `"698,965₩"` 처럼 텍스트로 저장돼 숫자 집계가 어려움.
   정제판인 `출발일` 탭을 정본으로 사용해야 함.
2. **`항공_최저가` ↔ `출발일` 스냅샷 불일치** — 같은 날짜인데 값이 다른 행 존재(예: 7/29, 7/14 등).
   서로 다른 수집 시점/소스로 추정. 정본 단일화 필요.
3. **에어부산 시장최저 미수집** — 우리요금만 있고 시장최저가 비어 있어 방어 판정 불가(`NO_MARKET`).
4. **수집 파이프 staleness** — 로그 탭에 `1일 전(stale)` + `⚠ 데이터 미갱신 — 수집 파이프 점검 필요`
   경보가 떠 있음(2026-06-20 기준 데이터가 06-19자).

> ⚠️ **요금 수집(스크래핑) 자체는 이 저장소 범위가 아니다.** 본 코드는 수집된 값을 받아
> 비교·방어 판정·리포트·신선도 점검을 수행하는 **모델/판정 계층**이다. 가격 수집은 정식 API
> (Amadeus/Duffel 등)로 별도 구성하는 것을 전제로 한다.

## 2. 코드가 재현한 결과 (검증)

`data/fares.csv`(시트 값 그대로 추출)로 방어를 재계산하면 **원본 시트의 ✅/🔴 표시와 정확히 일치**한다.

| 상품 | 방어성공 | 방어실패 | 우리요금 미입력 | 시장최저 미수집 | 방어율 | 실패분 합계 | 최대 실패분 |
| :-- | --: | --: | --: | --: | --: | --: | --: |
| 에어마카오 2박3일 | 19 | 29 | 86 | 0 | 40% | 2,420,992 | 172,882 |
| 에어마카오 3박4일 | 24 | 24 | 86 | 0 | 50% | 1,216,666 | 128,882 |
| 에어부산 2박3일 | 0 | 0 | 0 | 28 | 0% | 0 | 0 |
| 에어부산 3박4일 | 0 | 0 | 0 | 12 | 0% | 0 | 0 |

예) 에어마카오 2박3일 7/18 → 우리 956,000 > 시장 926,668 → `🔴 방어실패 -29,332` (시트와 동일).

> 운영 시사점: 7월말~8월 성수기에 **우리요금이 시장보다 비싼 구간이 다수**(2박3일 방어율 40%).
> 해당 구간 가격 재조정이 필요하다는 것을 코드가 수치로 보여준다.

## 3. 정확도 개선 (v2 — 적용 완료)

v1은 시트를 **그대로 재현**(이진 OK/실패)한다. 하지만 그 비교에는 정확도 한계가 있어 v2 계층을 추가했다.
**v1 결과는 그대로 보존**(시트 일치 보증)하고, v2는 같은 데이터를 더 정확하게 해석한다.

### 무엇이 부정확했나 → 무엇을 고쳤나

| # | 약점 | v2 개선 | 모듈 |
| - | :-- | :-- | :-- |
| 1 | **사과 vs 오렌지** — '시장최저'가 우리 항공편과 무관한 더 싼 편일 수 있음. 참고가(미운항)도 동급 취급 | 비교가능(comparable) 견적만으로 benchmark, 참고가는 `NON_COMPARABLE` 로 승패 제외 | `benchmark.py`, `verdict.py` |
| 2 | **2인가 = 1인최저 × 2 근사** — 실제 2석 발권가와 차이 → 거짓 실패 | 근사 여부(`approx_2p`)를 추적·명시 | `benchmark.py` |
| 3 | **단일 최저값의 취약성** — 글리치 1건이 benchmark 왜곡 | 2번째 최저 대비 **이상치 가드**(설정 비율↑면 2번째 채택) | `benchmark.py` |
| 4 | **이진 판정** — 근소차/과도하게 쌈 구분 못 함 | 🟦과방어/✅우위/🟨동률/🔴열위 **4단계 마진 판정** | `verdict.py`, `config.py` |
| 5 | **데이터 무검증** — 오래된/이상/누락 데이터를 그대로 신뢰 | STALE·SPIKE(고립 이상치)·NO_MARKET·REFERENCE_ONLY **자동 점검** | `validate.py` |

### 적용 효과(실데이터)

- **숨은 거짓 성공 적발**: v1에서 에어마카오 2박3일 "성공 19"였으나, 그중 **8건(7/10~7/17)은 우리 항공편 NX825 미운항 = 참고가**였다. v2는 이를 `비교불가`로 분리 → 실제 판정은 ✅우위 9 / 🟦과방어 2.
- **과방어 발견**: 시장보다 15만원 이상 싸게 판 구간을 `과방어`로 표시 → 가격 인상으로 회수 가능한 마진을 수치화(3박4일 과방어 여유분 합계 약 229만원).
- **데이터 건전성**: 전체가 06-19 단일 수집(2일 경과 STALE), 고립 이상치 1건(3박4일 7/13) 자동 검출. 성수기 연속 급등은 **오탐 없이** 통과.

> 핵심: v2는 **부정확한 입력을 그대로 신뢰하지 않는다.** "방어 성공/실패"라는 단정 대신, 비교 가능성과 데이터 신뢰도를 함께 보고한다.

```
macau-fare-monitor/

│   ├── config.py                  # 판정·검증 임계치(조정 가능)
│   ├── defense.py                 # v1: 방어 판정(시트 '방어' 열 재현)

│   ├── staleness.py               # 수집 신선도 로직
│   ├── loader.py                  # CSV → 도메인 객체(참고가 표식 파싱)
│   └── report.py                  # v1+v2 요약·상세·건전성 리포트
├── scripts/
│   └── build_report.py            # CLI: 데이터 → 리포트
└── tests/
    ├── test_defense.py            # v1 방어·신선도 로직
    ├── test_verdict.py            # v2 마진 판정
    ├── test_benchmark.py          # v2 benchmark/이상치 가드
    ├── test_validate.py           # v2 데이터 건전성/SPIKE
    └── test_data_integrity.py     # 데이터 정합성(134일 연속 등)
```

## 5. 실행 방법 (설치 불필요, Python 3.9+)

```bash
# 1) 테스트 — 32개 통과
PYTHONPATH=src python3 -m unittest discover -s tests -v

# 2) 리포트 생성(v1+v2+건전성) — 콘솔 요약 + reports/ 에 마크다운/CSV 저장
python3 scripts/build_report.py --as-of 2026-06-21 --out reports
```

라이브러리로 직접 사용:

```python
from macau_fare_monitor import evaluate, classify

# v1 (시트 재현)
evaluate(956000, 926668).label    # "🔴 방어실패 -29,332"

# v2 (마진 인지 판정)
classify(900000, 1000000).label   # "✅ 우위 +100,000"
classify(850000, 1050000).label   # "🟦 과방어 +200,000"  (가격 인상 검토)
```
