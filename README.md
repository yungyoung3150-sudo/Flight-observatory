# 마카오 항공 요금 — 컴플레인 가드레일 (macau-fare-monitor)

> **이 프로그램의 1차 목적**: 우리 공구가가 시장 최저가보다 **고객이 화낼 만큼(인당 10만원+)**
> 더 비싸지는 상황을 잡아내는 **가드레일**. 항공요금은 시시각각 변하므로 *완벽한 가격 매칭이
> 목표가 아니다.* 고객은 인당 1~2만원 차이는 용인하지만, OTA가 마케팅비를 태워 덤핑해 격차가
> 10만원 이상 벌어지면 컴플레인한다 — 그 노출을 막는 것이 존재 이유다.

예스트래블 마카오 공동구매 항공권 모니터링 로직을, 운영 중이던 구글 스프레드시트
(`마카오 항공 최저가 모니터링`)에서 추출해 **검토 가능한 코드**로 옮긴 뒤, 위 목적에 맞게 재설계했다.

- 원본 시트: 마카오 항공 최저가 모니터링 (예스트래블) — owner `miles@yestravel.co.kr`
- 수집 기준일: 2026-06-19 / 모니터링 구간: **2026-06-20 ~ 2026-10-31 (134일)**
- 대상 상품: 에어마카오 2박3일·3박4일, 에어부산 2박3일·3박4일

## 1. 원본 스프레드시트 분석

시트는 7개 탭으로 구성된 **"수집 → 비교 → 방어 판정 → 경보"** 파이프라인의 결과물이다.

| 탭 | 역할 | 핵심 컬럼 |
| :-- | :-- | :-- |
| `항공_최저가` | 에어마카오 시장 최저가(1인) 원시 수집 | 2박3일·3박4일 최저(1인), 수집일 |
| `에어마카오_2박3일` | **방어 시트** | 우리요금(2인), 시장최저(2인), 방어, 비고 |
| `에어마카오_3박4일` | **방어 시트** | 〃 |
| `에어부산_2박3일` | 방어 시트(시장최저 수집 예정) | 우리요금(2인)만 일부 입력 |
| `에어부산_3박4일` | 방어 시트(시장최저 수집 예정) | 〃 |
| `출발일`(통합) | 정제된 시장 최저가(1인) ISO 날짜판 | 2박3일·3박4일 최저(1인), 비고 |
| 수집 로그 | 수집 신선도 경보 | 수집시각, 데이터 신선도, `⚠ 데이터 미갱신` |

### 핵심 비즈니스 로직 — '방어'

각 출발일에 대해, 우리 공동구매가가 시장 최저가보다 **싸거나 같으면 방어 성공**, 비싸면 실패.
모든 비교는 **2인 합산가(KRW)** 기준이며, `시장최저(2인) = 시장 최저가(1인) × 2` 로 산출돼 있다.

```
gap = 시장최저(2인) - 우리요금(2인)
  gap >= 0  →  ✅ OK            (방어 성공, gap = 여유분)
  gap <  0  →  🔴 방어실패 {gap}  (예: "🔴 방어실패 -29,332", 부족분 = -gap)
  우리요금 또는 시장최저가 없으면 → 빈 칸(판정 불가)
```

### 발견한 데이터 품질 이슈 (CPO 검토 참고)

1. **`항공_최저가` 탭의 3박4일 열 포맷 깨짐** — `"698,965₩"` 처럼 텍스트로 저장돼 숫자 집계가 어려움.
   정제판인 `출발일` 탭을 정본으로 사용해야 함.
2. **`항공_최저가` ↔ `출발일` 스냅샷 불일치** — 같은 날짜인데 값이 다른 행 존재(예: 7/29, 7/14 등).
   서로 다른 수집 시점/소스로 추정. 정본 단일화 필요.
3. **에어부산 시장최저 미수집** — 우리요금만 있고 시장최저가 비어 있어 방어 판정 불가(`NO_MARKET`).
4. **수집 파이프 staleness** — 로그 탭에 `1일 전(stale)` + `⚠ 데이터 미갱신 — 수집 파이프 점검 필요`
   경보가 떠 있음(2026-06-20 기준 데이터가 06-19자).

> ⚠️ **요금 수집(스크래핑) 자체는 이 저장소 범위가 아니다.** 본 코드는 수집된 값을 받아
> 비교·방어 판정·리포트·신선도 점검을 수행하는 **모델/판정 계층**이다. 가격 수집은 정식 API
> (Amadeus/Duffel 등)로 별도 구성하는 것을 전제로 한다.

## 3. 가드레일 모델 (목적에 맞춘 재설계)

처음엔 시트를 그대로 옮기며 *대칭* 판정(싸다=좋음/비싸다=나쁨)을 넣었다. 하지만 이 프로그램의
1차 목적은 "**고객이 화낼 만큼 우리가 더 비싼 것**"을 막는 것이라, 판정을 **비대칭·인당 임계 경보**로
재설계했다. (설계는 3-렌즈 패널 + 2-검증 적대적 리뷰로 도출.)

### 핵심 모델

```
초과액(인당) = (우리요금_2인 − 시장최저_2인) ÷ 2      # 양수 = 우리가 더 비쌈

  초과액 ≤ +20,000/인          → 🟢 안전   (더 싸거나 살짝 비쌈 — 고객 용인)   무알림
  +20,000 < 초과액 < +100,000  → 🟡 주의   (사람이 점검, 판매결정 보류)        침묵 경보
  초과액 ≥ +100,000/인         → 🔴 경보   (컴플레인 위험 → 즉시 가격재산정/보류) 사이렌
```

- **비대칭**: '우리가 더 쌈'은 경보가 아니다 — 마진 사안(가격 인상 검토)일 뿐 별도 정보로만.
- **인당 기준**: 고객 분노 임계가 인당으로 진술됨(1~2만 용인, 10만+ 분노). 2인합산÷2로 환산하며,
  이 과정이 시트의 `1인×2` 근사를 되돌려 정확도도 올린다.
- **건전성 게이트 우선**: 시장최저 미수집(에어부산)은 **'안전'이 아니라 `수동확인 큐`** —
  과거 700k 덤핑이 난 BX 레그이고 Amadeus도 커버 못 하므로 절대 초록 처리 금지.
- **비교불가 제외**: 참고가/미운항(NX825)은 사과 vs 오렌지라 승패에서 분리.

### 실데이터 결과 (2026-06-19 사전 스냅샷)

| 상품 | 🔴경보 | 🟡주의 | 🟢안전 | 비교불가 | 수동확인 | 최대초과(인당) |
| :-- | --: | --: | --: | --: | --: | --: |
| 에어마카오 2박3일 | 0 | 23 | 17 | 8 | 0 | 86,441 |
| 에어마카오 3박4일 | 0 | 15 | 33 | 0 | 0 | 64,441 |
| 에어부산 2박3일 | 0 | 0 | 0 | 0 | 28 | — |
| 에어부산 3박4일 | 0 | 0 | 0 | 0 | 12 | — |

- **경보 0건**: 최악도 인당 86,441 < 10만. 사전 스냅샷에 컴플레인 임계 초과는 없다.
  → 이 도구의 진짜 값은 **판매 기간 중 OTA 덤핑**이 터질 때다.
- 8건(7/10~7/17)은 NX825 미운항 참고가 → `비교불가`로 정확히 제외.
- 에어부산 40건은 시장최저 공백 → `수동확인 큐`(안전 아님).

### 한계 — 정직하게

- **덤핑 '속도' 탐지는 단일 스냅샷으로 불가능**. 덤핑은 본질이 시간축(급락)·채널축(특정 OTA가
  published 운임 아래로)이라, **시계열 + 소비자 OTA 채널 데이터**가 있어야 잡힌다. 현재는 정적
  격차 라벨링까지만. (코드의 `dumping_suspect`는 벤치마크 이상치가 있을 때만 켜지는 약신호다.)
- 임계값(2만/10만)은 **시작 기본값**. 1주 갭 캘리브레이션(탑 SKU 매일 Amadeus/캐시/네이버 전결제·카드최저)으로 카드할인 버퍼를 실측해 보정해야 한다.

## 5. 실행 방법 (설치 불필요, Python 3.9+)

```bash
# 1) 테스트 — 39개 통과
PYTHONPATH=src python3 -m unittest discover -s tests -v

# 2) 리포트 생성 — 콘솔 경보 요약 + reports/ 에 마크다운/CSV 저장
python3 scripts/build_report.py --as-of 2026-06-21 --out reports
```

라이브러리로 직접 사용:

```python
from macau_fare_monitor import classify

classify(1_040_000, 1_000_000).label   # "🟢 안전 (인당 +20,000)"      용인
classify(1_100_000, 1_000_000).label   # "🟡 주의 (인당 +50,000)"      점검
classify(1_300_000, 1_000_000).label   # "🔴 경보 (인당 +150,000)"     컴플레인 위험
classify(900_000, None).label          # "⚪ 시장미수집(수동확인)"      안전 아님
```
